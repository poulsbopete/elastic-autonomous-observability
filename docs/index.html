Lab 2

Explore Telemetry with ES|QL

ES|QL is Elastic's next-generation piped query language. Use it to slice through millions of events instantly.

// Count recent errors by service
FROM logs-*
| WHERE @timestamp > NOW() - 15 MINUTES
| WHERE log.level == "ERROR"
| STATS error_count = COUNT(*) BY service.name
| SORT error_count DESC
| LIMIT 10
📈 Time Series
Query log and metric trends with BUCKET() aggregations. See error rate over time.
🗺️ APM Service Map
Visualize distributed traces across all 9 services and 3 cloud providers.
🏗️ Infrastructure
3 simulated hosts (AWS, GCP, Azure) with CPU, memory, and disk metrics.
ES|QL supports pipes (|), aggregations (STATS), filters (WHERE), and window functions. It runs at Elasticsearch speed — no sampling, full fidelity. Encourage learners to modify the queries and experiment.

Service Level Objectives

21 Auto-Generated SLOs

Three SLO types are created automatically per service — giving you a health baseline before any fault is injected.

💚
Availability
Measures the percentage of successful requests per service. Target: 99%+ uptime.
⚠️
Error Rate
Tracks the ratio of error-level log events. Fires when error ratio exceeds threshold.
⏱️
Latency
P95 response time target. Detects performance degradation before users notice.
Tip: Navigate to Observability → SLOs and set the time range to Last 15 minutes to see live burn rates from the running scenario.
SLOs are the bridge between raw telemetry and business health. When an SLO burns, the AI agent automatically factors it into root cause analysis.
